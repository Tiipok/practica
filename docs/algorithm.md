# Алгоритм проведения эксперимента

## 1. Подготовка

1. Создать тестовый ZIP-архив с известным паролем и типом шифрования
2. Задать алфавит и длину пароля
3. Определить размер парольного пространства: `|алфавит|^длина`

## 2. Запуск перебора

1. Определить число потоков (auto = hardware_concurrency)
2. Разделить пространство на равные диапазоны
3. Для каждого потока:
   - Создать копию генератора с диапазоном [start, end)
   - Создать экземпляр PasswordChecker
   - Итеративно получать следующий пароль, проверять
   - При нахождении: установить atomic-флаг → все потоки останавливаются
4. Дождаться завершения всех потоков (join)

## 3. Сбор метрик

- Время выполнения (std::chrono::high_resolution_clock)
- Количество проверенных паролей (atomic счетчик)
- Скорость = total_checks / elapsed_seconds
- Потребление памяти (Mach task_info)
- Загрузка CPU (Mach thread_info)

## 4. Сохранение

- Запись в SQLite (experiments.db)
- Экспорт CSV (results.csv)

## 5. Анализ

- Влияние длины пароля на время перебора
- Влияние размера алфавита на производительность
- Масштабируемость при увеличении числа потоков
- Сравнение ZipCrypto vs AES-256
- Сравнение Debug vs Release сборки

## Расчёт стойкости

Размер парольного пространства:
```
S = |A|^L
```
где |A| — размер алфавита, L — длина пароля.

Ожидаемое время полного перебора:
```
T = S / V
```
где V — скорость перебора (проверок/сек).

Пример (digits, L=5, V≈100,000):
```
S = 10^5 = 100,000
T ≈ 1 секунда
```

Пример (alphanum, L=5, V≈100,000):
```
S = 36^5 ≈ 60,466,176
T ≈ 605 секунд ≈ 10 минут
```

## Рекомендации по стойким паролям

1. Длина ≥ 8 символов (для реальной защиты)
2. Комбинация букв (разного регистра), цифр и спецсимволов
3. Избегать словарных слов
4. Для учебных целей: alphanum L=5 требует ~10 минут на Apple M4
